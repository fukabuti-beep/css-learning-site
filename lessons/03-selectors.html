セレクター完全ガイド

CSSセレクターは「どの要素にスタイルを適用するか」を指定します。基本から属性・疑似クラスセレクターまで、詳細度の計算方法も含めて完全に理解しましょう。

セレクターとは

セレクターは「どのHTML要素にスタイルを適用するか」を指定するCSSの構文です。適切なセレクターを使いこなすことで、HTMLを変更せずに柔軟なスタイリングが可能になります。

基本セレクター

CSS
/* タグセレクター：すべてのp要素 */
p { color: #666; }

/* クラスセレクター：class="card"の要素 */
.card { background: white; }

/* IDセレクター：id="header"の要素（ページ内で一意） */
#header { height: 64px; }

/* ユニバーサルセレクター：すべての要素 */
* { box-sizing: border-box; }
🔬 セレクターの種類を見分けるポイント
書き方
意味・対象
p
タグセレクター — HTMLの <p> タグすべてに適用。記号なし・英単語のみ
.card
クラスセレクター — ドット（.）が目印。class="card" がついた要素に適用。複数要素に使える
#header
IDセレクター — シャープ（#）が目印。id="header" がついた唯一の要素に適用
*
ユニバーサルセレクター — アスタリスク（*）。ページ内のすべての要素に適用
💡
HTMLと対応させて考えると理解しやすい：
<p class="card" id="header"> という要素には、p・.card・#header の3つすべてが適用されます

結合子セレクター

CSS
/* 子孫結合子（スペース）：nav内のすべてのaリンク */
nav a { color: white; }

/* 子結合子（>）：直接の子要素のみ */
ul > li { list-style: none; }

/* 隣接兄弟結合子（+）：直後の兄弟要素 */
h2 + p { font-size: 1.1rem; }

/* 一般兄弟結合子（~）：後続のすべての兄弟要素 */
h2 ~ p { color: #666; }

/* グループセレクター（,）：複数要素に同じスタイル */
h1, h2, h3 { font-weight: 700; }
📖 結合子の読み方チートシート
記号
意味と例
A 　 B （スペース）
A の中にあるすべての B。nav a → ナビ内のすべてのリンク（深さ不問）
A > B
A の直接の子である B のみ。ul > li → ulの直接の子liのみ（孫liは除外）
A + B
A の直後に来る兄弟 B。h2 + p → h2の次の1つのpだけ
A ~ B
A の後に来るすべての兄弟 B。h2 ~ p → h2より後ろのpすべて
A , B
A にも B にも同じスタイルを適用。h1, h2, h3 → 3種類の見出しすべて

属性セレクター

CSS
/* 属性が存在する */
input[disabled] { opacity: 0.5; }

/* 属性値が完全一致 */
input[type="text"] { border: 1px solid #ccc; }

/* 属性値で始まる */
a[href^="https"] { color: green; }

/* 属性値で終わる */
a[href$=".pdf"] { color: red; }

/* 属性値を含む */
img[src*="thumbnail"] { width: 200px; }
📖 属性セレクターの記号の意味
1
[disabled] — その属性が存在する要素。値は問わない
2
[type="text"] — 属性値が ぴったり "text" と一致する要素
3
[href^="https"] — ^ は「始まり」。https で始まるリンク（= 安全なリンク）を選択する使い方が多い
4
[href$=".pdf"] — $ は「終わり」。PDFファイルへのリンクに自動的にアイコンをつけるときに便利

疑似クラスセレクター

CSS
/* ユーザーインタラクション */
a:hover   { text-decoration: underline; }
a:focus   { outline: 2px solid blue; }
a:active  { opacity: 0.7; }
a:visited { color: purple; }

/* 構造的疑似クラス */
li:first-child    { font-weight: bold; }
li:last-child     { border: none; }
li:nth-child(2)   { color: red; }      /* 2番目 */
li:nth-child(odd) { background: #f5f5f5; } /* 奇数行 */
p:not(.special)   { color: #666; }    /* .special以外 */
📖 疑似クラスの読み方
1
a:hover — 「a要素にマウスが乗ったとき」のスタイル。コロン（:）の後が「状態の名前」
2
a:focus — キーボードのTabキーでフォーカスが当たったとき。アクセシビリティに重要
3
li:nth-child(odd) — odd（奇数）/even（偶数）でリストの縞模様を作れる。テーブルの偶数行に背景色をつけるのに便利
4
p:not(.special) — not() は「〜以外」。.specialクラスがついていないp要素すべてに適用

詳細度（Specificity）

複数のCSSルールが同じ要素に適用される場合、詳細度の高いルールが優先されます。詳細度は3つの数値で表されます。

🎯 詳細度の計算

1

IDセレクター
#id

+
1

クラス・属性・疑似クラス
.class / [attr] / :hover

+
1

タグ・疑似要素
p / h1 / ::before

1,0,0 → #header { } — 最強

0,2,0 → .nav.active { } — 強

0,1,1 → .nav a { } — 中

0,0,1 → a { } — 弱

⚠️

!important の注意点

color: red !important; はすべてのルールより優先されますが、詳細度の設計を壊すため、最終手段としてのみ使用しましょう。
